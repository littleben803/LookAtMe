# 想恋爱 2.0 UI Style Guide

## 设计方向

- 风格：霓虹浪漫风。
- 氛围：深色、发光、浪漫、演唱会现场感。
- 重点：高对比、远距离可读、横屏展示友好。
- 禁止：默认蓝色系统按钮、普通系统表单风、页面间风格割裂、过度原生的设置 Demo 感。

## 高保真设计图

- 设计图路径：[docs/Design/high_resolution_ui_design.png](/Users/bytedance/repo/lark/LookAtMe/docs/Design/high_resolution_ui_design.png)
- 画布尺寸：1536 x 1024。
- 覆盖范围：主要页面、首页更多相关页面、LED 展示效果页、Pro/购买/空态等必要页面、UI Style Guide。
- 落地优先级：具体页面实现以高保真设计图为第一视觉依据；当本文 token 与图中视觉冲突时，优先保持图中布局密度、组件形态、层级和氛围。
- 阶段边界：阶段 0 只记录设计规范，不实现业务页面；后续页面开发必须先对照该设计图拆解组件和状态。

## 图面结构规范

高保真设计图按 5 个区块组织，后续开发应按区块拆分验收：

| 区块 | 内容 | 设计要求 |
| --- | --- | --- |
| 01 主要页面 | 首页、收藏、设置 | 作为首版主 Tab 的视觉基准，保持深色背景、粉紫霓虹标题、卡片化内容区和底部 Tab 样式一致 |
| 02 首页更多相关页面 | 更多功能入口、样式选择、模板中心、文字颜色、背景颜色、字体选择、设置面板 | 用作二级功能页基准，采用紧凑网格、分段筛选、色板矩阵、列表选择和底部/侧边设置面板 |
| 03 LED 效果页面 | 经典跑马灯、爱心飘落、生日祝福等展示状态 | 必须全屏深色背景、大字号发光文字、底部玻璃控制面板、圆形播放/暂停主控 |
| 04 其他必要页面/交互 | 新手引导、Pro 会员页、购买成功、空状态 | 必须保留情绪化插画/图标、强 CTA、成功反馈和空态回流按钮 |
| 05 UI Style Guide | 色彩、字体、字号、圆角、间距、图标、按钮、组件 | 作为基础组件和 token 的直接视觉参考 |

## 页面级视觉规范

### 首页

- 顶部使用沉浸式霓虹 Banner，品牌「想恋爱」为第一视觉焦点，右上角保留 Pro 入口。
- 输入区使用深紫卡片和粉紫描边，输入提示文案保持低对比，字数统计位于右下角。
- 场景入口使用 5 个等宽方形图标卡：演唱会、表白、生日、接机、更多；图标使用粉色、紫色、蓝色、黄色霓虹色。
- 热门模板使用小型胶囊按钮，允许双列排列，按钮背景为深紫，描边和文字使用粉紫高亮。
- 样式选择使用横向卡片，卡片内必须呈现 LED 预览效果，不使用纯文本占位。
- 主 CTA 固定为大号粉色胶囊按钮，右侧带圆形播放图标。
- 底部 Tab 使用深色玻璃背景，选中态为粉色图标和文字，未选中态为低饱和灰紫。

### 收藏页

- 收藏列表使用深紫描边卡片。
- 每条收藏左侧展示小型场景/样式图标，中间展示收藏标题、日期和样式摘要，右侧使用心形收藏状态。
- 顶部右侧保留编辑入口，使用粉色文字。
- 空状态必须与图 04 的空状态一致：中心心形轮廓图标、简短说明、粉色主按钮。

### 设置页

- 设置区块使用圆角深紫容器分组。
- 颜色设置行右侧使用圆形色点。
- 数值设置使用粉色滑块，右侧显示百分比或等级文本。
- 开关使用粉色开启态，关闭态为深紫/灰紫。
- 列表行右侧使用 chevron，版本号右对齐显示。

### 更多与选择类页面

- 更多功能入口使用 2 列网格卡片，每个卡片包含图标和短标题。
- 样式选择使用免费/Pro 分段筛选，卡片为 2 列网格，Pro 项右上角显示小型 Pro 标识。
- 模板中心使用顶部分类 Tab 和模板胶囊按钮组合，分类包括演唱会、表白、生日、接机等。
- 文字颜色和背景颜色页面使用 5 列色板矩阵，选中态要有明显描边或中心标记。
- 字体选择页使用列表卡片，当前选中字体右侧高亮或勾选。
- 展示设置面板使用紧凑表单：滑块、单选胶囊、开关、重置按钮，不采用系统默认 Form 外观。

### LED 展示页

- 主内容必须全屏沉浸，背景保持接近黑紫色，避免普通页面导航感。
- LED 文本使用超大字号，文字必须有多层 glow：文字本体、外发光、远距离柔光。
- 不同样式需要有可识别的视觉差异，例如心形飘落、暖黄色生日祝福、粉色跑马灯。
- 底部控制面板为深色玻璃容器，包含速度、大小滑块和方向/镜像/闪烁/颜色/更多等图标按钮。
- 播放/暂停主按钮使用粉色圆形悬浮按钮，优先放在右侧，不能遮挡主要文字。
- 单击显示/隐藏控制面板、双击暂停/继续等交互后续实现时必须保持现场使用的低干扰原则。

### Pro、购买成功与空态

- Pro 会员页使用深色背景、金色 Pro 标识、权益列表和强粉色 CTA。
- 购买成功页使用绿色圆形 check 图标，成功文案居中，主按钮回到使用路径。
- 空状态使用大号心形轮廓图标、低对比说明文字和粉色主按钮，不出现冷冰冰的系统空列表。

## 字体

首版不引入第三方字体文件，避免版权和包体风险。使用系统字体模拟不同风格。

| Token | 字号 | 字重 | Design | 用途 |
| --- | --- | --- | --- | --- |
| `largeTitle` | 36 | heavy | rounded | 品牌大标题 |
| `pageTitle` | 24 | bold | rounded | 页面标题 |
| `sectionTitle` | 18 | bold | rounded | 分区标题 |
| `body` | 15 | regular | rounded | 正文 |
| `caption` | 12 | regular | rounded | 辅助说明 |
| `button` | 17 | bold | rounded | 主按钮 |
| `ledDisplay` | 96/120 | heavy | rounded | LED 全屏展示 |

### 设计图字体补充

- 图稿普通中文界面字体标注为苹方 `PingFang SC`，工程实现默认使用系统中文字体即可。
- 图稿数字字体标注为 `DIN Alternate`，可用于字数统计、百分比、日期、价格等数字信息；若系统环境不可用，使用系统数字字体兜底。
- 图稿 LED 标题展示为更具装饰性的中文展示字风格。首版仍遵守“不引入第三方字体文件”的总规范，用系统 rounded/heavy 模拟；只有在确认字体授权和包体影响后，才允许接入真实字体文件。
- 字号遵循图稿标注：LED 标题 80/100/120，页面标题 22/24，模块标题 16/18，正文 14，辅助说明 12，标签/按钮 10/12。

## 间距

基础间距 token：

- `xxs`: 4
- `xs`: 8
- `sm`: 12
- `md`: 16
- `lg`: 20
- `xl`: 24
- `xxl`: 32
- `xxxl`: 40

布局规范：

- 页面横向边距：20
- 卡片内边距：16
- 分区间距：24
- 按钮内容横向间距：16
- 组件内部小元素间距：8 或 12
- 图稿实际页面采用高密度移动端布局：同屏信息量较高，但所有可点击区域必须保持可点按；不要为了“留白感”擅自拉大页面密度。

## 阴影与发光

- 卡片阴影：粉紫色低透明阴影，强调层级但不制造脏边。
- 霓虹发光：主色或电光蓝双层 glow，用于品牌标题、LED 字体和关键 CTA。
- 浮层阴影：黑色低透明投影，用于底部控制面板、弹层、Pro 卡片。
- 图稿中的描边普遍为 1px 级别低透明粉紫线，不应使用厚重边框。
- 卡片和面板的 glow 应只强调边缘和 CTA，不应把整页做成高亮发光块。
- LED 文本 glow 可以更强，但必须保证文字轮廓清晰，不能糊成不可读色块。

## 图标规范

- 图稿图标使用线性 SF Symbol 风格，线宽轻、中等，整体圆润。
- 图标色彩可使用粉色、紫色、蓝色、黄色，但同一组件组内应保持层级一致。
- 主要图标集合包括：演唱会、表白、生日、接机、更多、设置、收藏、返回、播放、暂停、编辑、删除。
- Tab 图标选中态为粉色，未选中态为灰紫；不要使用系统默认蓝色。
- 功能入口图标必须放在深紫卡片内，避免单独漂浮在背景上。

## 组件规范

### PrimaryButton

- 使用粉色到紫色霓虹渐变。
- 高度建议不低于 56。
- 圆角使用 `LookRadius.button`。
- 禁用态降低透明度，不出现系统蓝色。
- 可支持 SF Symbol 图标、loading 态和禁用态。
- 设计图包含两种主按钮尺寸：大号主按钮用于核心 CTA，中号主按钮用于购买/确认等局部 CTA。
- 主按钮右侧可内嵌白色圆形播放图标，首页「开始展示」必须使用该形态。

### SecondaryButton

- 使用透明或深紫背景，粉紫描边，粉色文字。
- 适用于「去首页」「稍后再说」「恢复购买」等次级动作。
- 高度低于主按钮，但圆角和描边风格必须一致。

### ProButton

- 使用深紫背景、粉紫描边和金色皇冠/Pro 文案。
- 用于首页右上角、Pro 入口、样式锁定状态。
- 不应压过主 CTA，但必须在视觉上容易识别。

### NeonCard

- 使用 `cardPurple` 或 `elevatedPurple` 背景。
- 使用细霓虹描边。
- 圆角使用 `LookRadius.card`。
- 内边距默认 16。
- 用于模板卡片、设置分组、收藏卡片等重复内容容器。
- 卡片背景允许轻微纵向渐变，但必须保持暗色基调。
- 样式卡片必须展示样式预览图，不允许仅写样式名。

### SectionHeader

- 标题使用 `sectionTitle`。
- 可选副标题使用 `caption`。
- 可选右侧操作位。
- 分区之间保持 24 间距。

### EmptyStateView

- 使用霓虹图标或 SF Symbol。
- 文案必须直接说明当前为空和下一步动作。
- 可选主按钮用于创建、添加、恢复或返回核心路径。
- 不作为业务入口假实现，只作为空态骨架组件。

### InputField

- 输入框使用深紫背景、20 圆角、1px 粉紫描边。
- 占位文字使用 `textTertiary` 或更低透明度。
- 字数统计右下角显示，数字建议使用 DIN Alternate。
- 输入框不使用系统 TextField 默认蓝色光标和默认边框样式；光标和选中态应使用 `primaryPink`。

### TemplateChip

- 背景为深紫或透明深色，描边为粉紫低透明。
- 选中态提高背景亮度和描边亮度。
- 文案短小，允许带 emoji 或场景小图标，但不能影响可读性。

### StyleCard

- 2 列或横向滚动卡片均可，卡片内必须包含 LED 样式预览。
- Pro 样式右上角显示小型 `Pro` 标识。
- 选中态使用粉色描边、轻微 glow 或角标，不使用系统勾选蓝色。

### Switch

- 开启态为粉色轨道或粉色圆点。
- 关闭态为深紫轨道和灰紫圆点。
- 设置页和展示设置面板必须统一该样式。

### Slider

- 进度轨道使用 `primaryPink`，未完成轨道使用灰紫。
- 圆形滑块为白色或粉白色，并带轻微粉色 glow。
- 数值文本右侧展示，速度、大小等现场调节项必须可快速识别。

## 阶段 0 组件边界

阶段 0 只实现 `PrimaryButton`、`NeonCard`、`SectionHeader`、`EmptyStateView` 的基础骨架。`NeonTextInput`、`TemplateChip`、`StyleCard`、`PaywallButton` 等后续在具体页面阶段按真实需求实现。
